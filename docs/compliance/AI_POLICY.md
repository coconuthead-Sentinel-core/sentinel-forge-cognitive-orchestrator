## 1. Purpose and Scope

This policy governs the design, development, operation, and use of artificial intelligence within the **Sovereign Forge** platform. It applies to Shannon Bryan Kelly (Architect/Owner) and any contributors or end users.

---

## 2. What Sovereign Forge Is

Sovereign Forge is a real-time, self-optimizing cognitive AI operating system powered by **Azure OpenAI o4-mini**. It features a CNO-AX Metacognition Engine, 14-Mirror Cognitive Array, voice interface, and persistent Sentinel Profile system. It is the most architecturally advanced platform in the Forge trilogy.

---

## 3. Acceptable Use

### ✅ Acceptable
- Cognitive research and self-optimization exploration
- Real-time AI-assisted knowledge synthesis
- Portfolio demonstration and enterprise architecture showcase
- Personal productivity and thought organization
- Performance benchmarking and AI system research

### ❌ Not Acceptable
- Manipulating users through subliminal or deceptive AI behavior
- Processing sensitive personal data without DPIA review
- Using voice interface to record conversations without consent
- Using platform outputs as professional clinical, legal, or financial advice
- Deploying as surveillance or behavioral monitoring system

---

## 4. AI Act Classification

**Classification: General-Purpose / Limited-Risk AI**

Sovereign Forge does not make consequential decisions about individuals, does not affect employment or safety, and does not use prohibited techniques. The CNO-AX Engine optimizes *system* performance — not individual scoring or profiling.

**Obligations met:**
- Users informed they interact with AI (agent names, mode displayed)
- No prohibited AI practices
- Documentation maintained (this policy + FRIA)
- Human oversight: all outputs advisory

---

## 5. Privacy and GDPR

- Voice input processed in-session only — not stored by default
- No persistent user data stored without Cosmos DB (not yet active)
- Sentinel Profile stores *cognitive preferences* — not personal identity data
- Azure processes audio/text under Microsoft data processing terms
- Users must not speak personal identifying information into voice interface without consent review

## 7. Voice Interface Policy

The TTS/STT Karaoke system introduces specific obligations:
- Users must be informed when voice recording is active
- Voice data is processed locally (STT) and via Azure (AI response)
- No voice recordings stored by default
- Voice interface must not be used to record third parties without consent

---

## 8. Incident Reporting

1. Document the input and output immediately
2. Log in `docs/compliance/INCIDENT_LOG.md`
3. Set `MOCK_AI=true` if needed
4. Review and remediate before re-enabling
